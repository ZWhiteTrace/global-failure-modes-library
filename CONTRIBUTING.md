# Contributing to Global Failure Modes Library

This project welcomes contributions from **all backgrounds** - you don't need to be an engineer.

## Who Should Contribute

- Healthcare professionals who've seen system failures
- Aviation experts with safety incident knowledge
- Finance professionals who understand risk failures
- Managers who've witnessed organizational breakdowns
- Anyone who has observed patterns of failure in their domain

## How to Contribute a Failure Mode

### 1. Use the Template

Create a new file in `modes/` following this structure:

```markdown
# [Failure Mode Name]

One-sentence description of the failure pattern.

## Warning Signs

- [ ] Observable symptom 1
- [ ] Observable symptom 2
- [ ] Observable symptom 3

## Real-World Examples

**[Industry/Context]**: Brief description of failure case.

## Mitigation Checklist

1. **Action Name**: Specific actionable step
2. **Action Name**: Specific actionable step

## Questions to Ask

- Question that helps identify this failure mode?
- Question that helps prevent this failure mode?

## Related Modes

- [Related Mode](./related-mode.md)
```

### 2. Guidelines

- **Be Specific**: Vague warnings are not actionable
- **Real Examples**: At least one concrete case, anonymized if needed
- **Actionable Mitigations**: Things someone can actually do
- **Cross-Reference**: Link to related failure modes

### 3. Submit

**GitHub Users**: Open a Pull Request

**Non-GitHub Users**:
- Email your contribution to [maintainer email]
- Or open an Issue with your content

## Adding Examples to Existing Modes

Found a new example of an existing failure mode? Add it under the Real-World Examples section via PR or Issue.

## Questions?

Open an Issue - no question is too basic.
